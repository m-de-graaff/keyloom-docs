# Introduction to Keyloom

**Modern, security-first authentication for JavaScript applications**

Keyloom is a comprehensive authentication library designed specifically for modern JavaScript and TypeScript applications. Built with security as the top priority, it provides a complete, production-ready solution for handling user authentication, session management, and authorization in web applications.

## What is Keyloom?

Keyloom is an open-source authentication system that combines the best practices of modern web security with an exceptional developer experience. Unlike traditional authentication solutions that require complex setup and configuration, Keyloom provides a batteries-included approach that gets you up and running quickly while maintaining enterprise-grade security standards.

<Callout type="info">
Keyloom is built for the modern web stack with first-class TypeScript support, edge runtime compatibility, and framework-agnostic design principles.
</Callout>

## Key Features

### 🔐 **Multiple Authentication Strategies**
- **OAuth 2.0/OIDC Providers**: GitHub, Google, Discord, Apple, Auth0, GitLab, Microsoft, X (Twitter)
- **Email/Password Authentication**: Secure credential-based login with Argon2id hashing
- **Magic Links**: Passwordless authentication via email verification
- **Multi-Factor Authentication**: TOTP and SMS-based 2FA support

### 🗄️ **Database Agnostic**
- **Multiple Adapters**: Prisma, Drizzle, PostgreSQL, MySQL, MongoDB, and in-memory storage
- **Custom Adapters**: Build your own adapter for any database or storage system
- **Contract Testing**: All adapters are thoroughly tested against a common contract

### ⚡ **Framework Support**
- **Next.js Integration**: First-class support for both App Router and Pages Router
- **Edge Runtime**: Full compatibility with Vercel Edge Runtime and Cloudflare Workers
- **Middleware Support**: Built-in route protection and authentication guards
- **Server Components**: Seamless integration with React Server Components

### 🛡️ **Security First**
- **CSRF Protection**: Double-submit cookie pattern with dedicated CSRF endpoints
- **Secure Sessions**: HttpOnly, Secure, and SameSite cookie configurations
- **Rate Limiting**: Built-in protection against brute force attacks
- **PKCE Support**: Proof Key for Code Exchange for OAuth flows
- **Token Security**: HMAC-SHA256 hashing for tokens at rest

### 📱 **Modern Development Stack**
- **TypeScript First**: Complete type safety across all packages and integrations
- **ESM/CJS Support**: Works with both module systems
- **Tree Shaking**: Optimized bundle sizes with selective imports
- **Developer Tools**: Comprehensive CLI for project setup and management

### 🔄 **Flexible Session Management**
- **JWT Sessions**: Stateless authentication with EdDSA/ES256 signing
- **Database Sessions**: Traditional server-side session storage with rolling sessions
- **Hybrid Approach**: Combine JWT and database sessions for optimal performance
- **Session Rotation**: Automatic token rotation for enhanced security

### 🚦 **Role-Based Access Control (RBAC)**
- **Organizations**: Multi-tenant support with organization-based access
- **Roles & Permissions**: Granular permission system with role inheritance
- **Team Management**: Built-in team and membership management
- **Policy Engine**: Flexible policy-based authorization

## Architecture Overview

Keyloom is built as a modular monorepo with clear separation of concerns:

```mermaid
graph TB
    A[Your Application] --> B[@keyloom/nextjs]
    A --> C[@keyloom/core]
    
    B --> C
    C --> D[@keyloom/adapters]
    C --> E[@keyloom/providers]
    
    D --> F[Prisma]
    D --> G[Drizzle]
    D --> H[PostgreSQL]
    D --> I[MongoDB]
    
    E --> J[GitHub]
    E --> K[Google]
    E --> L[Discord]
    E --> M[Apple]
    
    N[@keyloom/cli] --> O[Project Setup]
    N --> P[Migrations]
    N --> Q[Code Generation]
```

### Core Packages

- **`@keyloom/core`** - Core authentication logic, cryptography, session management, and RBAC
- **`@keyloom/nextjs`** - Next.js integration with App/Pages Router support and middleware
- **`@keyloom/server`** - Standalone server implementation with Fastify integration
- **`@keyloom/cli`** - Command-line tools for project initialization and management

### Extension Packages

- **`@keyloom/adapters`** - Database adapters for various storage systems
- **`@keyloom/providers`** - OAuth provider implementations
- **`@keyloom/ui`** - Pre-built UI components for authentication flows
