# Installation Guide

This guide covers all the ways to install and set up Keyloom in your project.

## Prerequisites

Before installing Keyloom, ensure you have the following:

- **Node.js** >= 18.17.0
- **TypeScript** >= 5.4 (recommended)
- A package manager: **npm**, **pnpm**, or **yarn**

<Callout type="info">
  Keyloom is built with TypeScript and provides full type safety. While you can
  use it with JavaScript, TypeScript is highly recommended for the best
  developer experience.
</Callout>

### Memory Adapter (Development Only)

The memory adapter is included with `@keyloom/core` and requires no additional installation.

<Callout type="warn">
  The memory adapter should only be used for development and testing. It stores
  data in memory and will lose all data when the process restarts.
</Callout>

## Manual Installation from Source

If you need to install from source or contribute to Keyloom:

```bash
# Clone the repository
git clone https://github.com/m-de-graaff/keyloom.git
cd keyloom

# Install dependencies
pnpm install -w

# Build all packages
pnpm build

# Link packages locally (optional)
pnpm link --global @keyloom/core
pnpm link --global @keyloom/nextjs
pnpm link --global @keyloom/cli
```

## Verification

After installation, verify everything is working:

1. **Check package versions:**

   ```bash
   npm list @keyloom/core
   npm list @keyloom/nextjs
   ```

2. **Test CLI installation:**

   ```bash
   keyloom --help
   ```

3. **Create a test configuration:**

   ```typescript
   // test-keyloom.ts
   import { memoryAdapter } from "@keyloom/core";

   const config = {
     adapter: memoryAdapter(),
     session: { strategy: "database" as const },
     secrets: { authSecret: "test-secret" },
   };

   console.log("Keyloom configuration created successfully!");
   ```

## Troubleshooting

### Common Issues

#### TypeScript Errors

If you encounter TypeScript errors, ensure you have the correct version:

```bash
npm install typescript@^5.4
```

#### Peer Dependency Warnings

Install missing peer dependencies:

```bash
# For Next.js integration
npm install next@^13 react@^18

# For TypeScript support
npm install typescript@^5.4
```

#### Module Resolution Issues

If you encounter module resolution issues, check your `tsconfig.json`:

```json
{
  "compilerOptions": {
    "moduleResolution": "node",
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true
  }
}
```
