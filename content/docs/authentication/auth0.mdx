Authenticate with Auth0 tenants (custom domains supported).

<Steps>

### 1) Auth0 application

- Application type: Regular Web App
- Allowed Callback URLs: `https://YOUR_DOMAIN/api/auth/callback/auth0`
- Copy Domain, Client ID, Client Secret

```bash
# .env.local
AUTH0_DOMAIN=your-tenant.us.auth0.com
AUTH0_CLIENT_ID=...
AUTH0_CLIENT_SECRET=...
```

### 2) Configure provider

```ts
// keyloom.config.ts
import { PrismaAdapter } from '@keyloom/adapters'
import { PrismaClient } from '@prisma/client'
import { Auth0 } from '@keyloom/providers'

const db = new PrismaClient()
export default {
  adapter: PrismaAdapter(db),
  providers: [
    Auth0({
      domain: process.env.AUTH0_DOMAIN!,
      clientId: process.env.AUTH0_CLIENT_ID!,
      clientSecret: process.env.AUTH0_CLIENT_SECRET!,
    }),
  ],
}
```

### 3) UI

## Troubleshooting

- Domain must include your tenant domain (or custom domain)
- Callback URL mismatch is the most common cause of 4xx responses
