# UI Integration

Keyloom does not force a UI. Compose your own using React components and the hooks provided by `@keyloom/nextjs`.

## Read the session

```tsx
import { getServerSession } from "@keyloom/nextjs";
import config from "@/keyloom.config";

export default async function Layout({ children }: { children: React.ReactNode }) {
  const session = await getServerSession(config);
  return (
    <>
      <NavBar user={session?.user} />
      {children}
    </>
  );
}
```

## Sign in and out

Use simple links that hit the handler endpoints.

```tsx
<a href="/api/auth/signin/github">Continue with GitHub</a>
<form method="post" action="/api/auth/logout">
  <button type="submit">Sign out</button>
</form>
```

## Guard components by role

```tsx
function AdminOnly({ children }: { children: React.ReactNode }) {
  const { data: session } = useClientSession();
  if (!session || session.org?.role !== "admin") return null;
  return <>{children}</>;
}
```

## Patterns

- Use server components to read session and pass it down via props or context
- Keep provider buttons as plain anchors to the signin endpoint
- Build forms for username or magic-link flows against your own custom routes that call Keyloom server APIs
