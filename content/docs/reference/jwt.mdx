# JWT Strategy

When `session.strategy` is set to `"jwt"`, Keyloom issues access/refresh tokens. On Next.js, use @keyloom/nextjs helpers for verification.

## Server helpers
```ts
import { createJwtConfig, getJwtSession, requireJwtAuth } from "@keyloom/nextjs/jwt-server";

const jwtConfig = createJwtConfig(process.env as any);
const { user, session } = await getJwtSession(jwtConfig);
// or
await requireJwtAuth(jwtConfig);
```

Required env:
- KEYLOOM_JWT_JWKS_URL: JWKS endpoint serving public keys
- KEYLOOM_JWT_ISSUER: Issuer (should match your baseUrl)

<Callout>
The helpers automatically read cookies or Authorization headers. To customize, pass a NextRequest to the functions.
</Callout>
