### 2) Build the registration page

<Tabs>
<Tab value="app-router" title="App Router">

```tsx
"use client"
import { Card } from '@keyloom/ui/components/card'
import { SignUpForm } from '@keyloom/ui/auth'

export const metadata = { title: 'Create account' }
export default function Page() {
  return (
    <div className="min-h-dvh grid place-items-center p-6">
      <Card className="w-full max-w-md p-6">
        <h1 className="text-2xl font-semibold mb-4">Create your account</h1>
        <SignUpForm redirectTo="/onboarding" />
      </Card>
    </div>
  )
}
```

</Tab>

### 3) Server-side validation

- Enforce password strength in your backend: minimum length, character classes, and breached password checks if required.
- Keyloom performs CSRF validation for POST endpoints out of the box.

---

### 4) Email verification (optional)

If enabled, users receive a verification link via email. Provide a verification page using `EmailVerificationForm` from `@keyloom/ui/auth`.

---

### 5) Troubleshooting

- Duplicate email: The adapter returns a unique constraint error; show a user-friendly message.
- 403 on register: Ensure CSRF token and cookie are present.
- Emails not sent: Verify SMTP credentials and that your server can connect to the mail host.

</Steps>
