A complete, production-ready password reset request ("forgot password") flow using Keyloom UI and Next.js.

<Callout>
Keyloom provides built-in endpoints for password flows via the API handler; no custom route wiring is required beyond creating the handler.
</Callout>

## Steps

<Steps>

### 1) Ensure API handler is set up

```ts
// App Router: app/api/auth/[[...keyloom]]/route.ts
import { createNextHandler } from '@keyloom/nextjs'
import config from '@/keyloom.config'
export const { GET, POST } = createNextHandler(config)
```

```ts
// Pages Router: pages/api/auth/[...keyloom].ts
import { createPagesApiHandler } from '@keyloom/nextjs'
import config from '@/keyloom.config'
export default createPagesApiHandler(config)
```

### 2) Add the Forgot Password page

<Tabs>
<Tabs.Tab title="App Router">

```tsx
// app/(auth)/forgot-password/page.tsx
"use client"
import { Card } from '@keyloom/ui/components/card'
import { ForgotPasswordForm } from '@keyloom/ui/auth'

export const metadata = { title: 'Forgot password' }
export default function Page() {
  return (
    <div className="min-h-dvh grid place-items-center p-6">
      <Card className="w-full max-w-md p-6">
        <h1 className="text-2xl font-semibold mb-4">Reset your password</h1>
        <ForgotPasswordForm />
      </Card>
    </div>
  )
}
```

</Tabs.Tab>
<Tabs.Tab title="Pages Router">

```tsx
// pages/forgot-password.tsx
import dynamic from 'next/dynamic'
const ForgotPasswordForm = dynamic(() => import('@keyloom/ui/auth').then(m => m.ForgotPasswordForm), { ssr: false })
export default function Page() {
  return (
    <div className="min-h-screen grid place-items-center p-6">
      <div className="w-full max-w-md">
        <h1 className="text-2xl font-semibold mb-4">Reset your password</h1>
        <ForgotPasswordForm />
      </div>
    </div>
  )
}
```

</Tabs.Tab>
</Tabs>

### 3) What the form does

- POST /api/auth/forgot-password
- Displays a success message when the reset link is sent
- Provides error messages for failed requests

</Steps>

## Troubleshooting

- CSRF errors on POST: ensure you used the Keyloom API handler; do not bypass middleware for auth endpoints.
- Email not received: verify your email delivery configuration/provider in your Keyloom setup; check logs.
- 404 on endpoint: verify your API handler path matches the code above.
