# CLI Concepts

The Keyloom CLI provides initialization, route scanning, migrations, diagnostics, and UI scaffolding.

## Commands overview

```bash
keyloom init [--session <database|jwt>] [--adapter <prisma|drizzle-pg|drizzle-mysql|postgres|mysql2|mongo>] [--providers <csv>] [--rbac] [--cwd <dir>] [-y|--yes]
keyloom migrate [--cwd <dir>] [--dry-run] [--force]
keyloom routes [--out <file>] [--outJson <file>] [--cwd <dir>]
keyloom doctor [--json] [--strict] [--cwd <dir>]
keyloom generate migration
keyloom ui setup
keyloom ui add <sign-in|user-button|org>
```

## keyloom init
- Creates keyloom.config.(ts|js)
- Adds a Next.js API handler (App or Pages Router)
- Adds middleware with generated routes mapping
- Writes .env.example with provider variables
- Generates adapter-aware migration artifacts

## keyloom migrate
- Detects adapter and runs the right migration command or prints next steps

## keyloom routes
- Scans your code for annotated route visibility and writes a generated routes file

## keyloom doctor
- Runs a set of project checks
- With --json it prints machine-readable output

## keyloom ui
- setup: configures Tailwind preset + CSS vars
- add sign-in | user-button | org: scaffolds UI primitives
