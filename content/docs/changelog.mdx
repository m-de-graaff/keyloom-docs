## Release Process

### Version Numbering

We follow [Semantic Versioning](https://semver.org/):

- **MAJOR** version for incompatible API changes
- **MINOR** version for backwards-compatible functionality additions
- **PATCH** version for backwards-compatible bug fixes

### Release Schedule

- **Major releases**: Every 6-12 months
- **Minor releases**: Every 4-8 weeks
- **Patch releases**: As needed for critical fixes

### Breaking Changes

Breaking changes are clearly marked and include:

- Migration guides
- Deprecation warnings in previous versions
- Detailed upgrade instructions

### Security Updates

Security updates are released immediately and include:

- CVE numbers when applicable
- Detailed security advisories
- Upgrade recommendations

## Migration Guides

### Upgrading to v1.6.0

<Callout type="info">
  This release includes new RBAC features and server package. No breaking
  changes for existing users.
</Callout>

**New Features:**

- RBAC system is opt-in via configuration
- Server package is a new standalone option
- CLI doctor command helps diagnose issues

**Recommended Actions:**

1. Update all Keyloom packages to v1.6.0
2. Run `npx keyloom doctor` to check your setup
3. Consider enabling RBAC if you need role-based access control

### Upgrading to v1.5.0

<Callout type="warn">
  This release includes breaking changes to the adapter interface.
</Callout>

**Breaking Changes:**

- Adapter interface updated for better type safety
- Session schema changes require database migration
- Configuration format updated

**Migration Steps:**

1. Update configuration format:

   ```typescript
   // Before
   export default {
     database: { adapter: prismaAdapter(prisma) },
   };

   // After
   export default {
     adapter: prismaAdapter(prisma),
   };
   ```

2. Run database migration:

   ```bash
   npx keyloom migrate
   ```

3. Update custom adapters to new interface

### Upgrading to v1.4.0

**New Features:**

- OAuth providers now available
- CLI tool for easier setup

**Recommended Actions:**

1. Install CLI globally: `npm install -g @keyloom/cli`
2. Initialize new projects with: `npx keyloom init`

### Upgrading to v1.3.0

**Breaking Changes:**

- Password hashing changed to Argon2id
- Session format updated

**Migration Steps:**

1. Existing passwords need to be rehashed on next login
2. All existing sessions will be invalidated
